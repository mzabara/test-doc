# Тестовое задание

Angular-приложение для просмотра документов и добавления аннотаций.

## Архитектура приложения
- Feature слой: pages/viewer, pages/page, pages/annotation - UI.
- Data слой: shared/api + DocumentStore — получение документа.
- State/UI слой: AnnotationStore, ZoomStore — состояние интерфейса и аннотаций.
- Shared/UI: shared/components, shared/directives, shared/utils.

## Плюсы реализации
- Standalone компоненты и ленивые маршруты для масштабирования.
- Состояние документа, аннотаций и zoom вынесено в отдельные store на сигналах.
- Перетаскивание реализовано без библиотек (pointer events).
- `NgOptimizedImage` для статичных изображений страниц.
- Реактивные формы для ввода текста аннотаций.

## Минусы реализации
- Нет сохранения на сервер: аннотации живут только в памяти.
  - Решение: добавить API для `GET/POST` аннотаций и сохранять их по `documentId`.
- Нельзя редактировать текст уже созданной аннотации.
  - Решение: добавить режим редактирования по клику и отдельные действия "Сохранить/Отмена".
- Нет виртуализации длинных документов.
  - Решение: добавить рендер по мере скролла (виртуальный список) при большом числе страниц.
- Жестко задан дефолтный `id` в маршруте (`/viewer/view/1`).
  - Решение: убрать дефолтный `id` или получать его извне.
